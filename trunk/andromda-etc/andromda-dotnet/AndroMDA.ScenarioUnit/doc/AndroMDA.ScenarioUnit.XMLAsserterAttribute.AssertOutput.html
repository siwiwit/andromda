Remarks

This method is called by the TestScenarioHelper class once for every return value/output parameter of a test method for which this asserter is configured.

This method serializes the actual test output returned from the system under test using the System.Xml.Serialization.XmlSerializer class.

This method also saves the actual output to a physical file, so that it can be reviewed manually after the test case is run. This file is created in the folder specified by the ActualOutputDir property.

The name of the physical file is created using the same naming convention as that used by the XMLDataProvider class: methodName_scenarioName_parameterName.xml.

Note    In case of return parameters, there is no "parameterName", so, the file name is just methodName_scenarioName.xml.

This method reads expected output from XML files from the location specified by the ExpectedOutputDir property.

The file naming convention used to identify the file to be read is the same here as well: methodName_scenarioName_parameterName.xml.

Not all values in actual and expected output can be compared in some situations.

For example, the output may contain a generated ID from the database or a dateTime field that is filled with the current date time value by the system under test.

Such parts that cannot be compared need to be ignored while asserting the actual output against expected output. This can be done by creating a rules file for the specific return value/output parameter for each test scenario.

This rules file takes the form of an XML file with XPath queries identifying the nodes in the actual and expected output XML files that need to be ignored. This rules file also follows the same naming convention of methodName_scenarioName_parameterName.xml and is placed in the directory specified by the RulesDir property.
